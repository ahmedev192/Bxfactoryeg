Concept model

Stage · a unit of work (Cut, Sew, Print, QC…). Defined once in the Stages library.
Step · one slot in a workflow. A step contains one stage (sequential) OR multiple stages (parallel — they all run concurrently).
Workflow template · ordered list of steps. Steps run in sequence; stages within a step run in parallel.
Process · a real resource performing work at one stage. Has 3-point time estimates, cost, confidence %, splittable flag.
Order · routes through one workflow. For each stage you (a) decide whether the stage applies, (b) pick candidate processes, (c) set quantity shares if splitting.

Time model (PERT + Monte Carlo + quantity thresholds)

Each process has 3-point estimates (optimistic / most-likely / pessimistic). The confidence % shrinks the spread when sampling. For each scenario, N Monte Carlo trials draw from triangular distributions.

Within a step: step duration = MAX(stage durations) — slowest parallel stage wins.
Within a stage with splits: stage duration = MAX(branch durations) — slowest split branch wins.
Total project duration: SUM(step durations) — steps are sequential.

Quantity thresholds (NEW v4): each process can declare extra days when quantity exceeds set thresholds. The qty that drives the bump is the branch quantity — i.e., the share assigned to that process if split, or the full order qty if not. The highest matching threshold wins (no stacking). Example: Factory A has +2d at 1000 pcs, +5d at 3000 pcs. If a 4000-pc order goes 100% to A, time = base + 5d. If split 60/40, A gets 2400 pcs (+2d bump), B gets 1600 pcs (its own threshold table applies).

Note on split quantities: per your config, time does NOT scale linearly with quantity. Each branch takes its base time PLUS any threshold bump. Cost DOES scale with quantity share.

Excel export — simple action format

After running the planner, click 📊 Export this run to Excel on the Scenarios tab. The .xlsx contains:

• Order Brief — quantity, deadline, workflow, with blanks to fill in order number and brand name
• Scenarios — bullet-point comparison, recommended scenario marked with ⭐
• Action Plan — Scenario N (one per scenario) — step-by-step list with stage start/deadline dates, "Done?" checkbox, and "Actual completion date" column for tracking actuals

Stage deadlines are calculated as cumulative P95 from Monte Carlo — the date each stage must finish to give 95% confidence in meeting that step's deadline. Confidence levels you set per process feed directly into these dates.

Honest limitations

• Steps are strictly sequential between each other; only stages within a step are parallel.
• Costs are deterministic (only times have distributions).
• Resource conflicts across multiple in-flight orders are not modeled.
• Confidence % is subjective — replace with empirical std once you have ≥30 historical observations per process.
• Per-step P95s don't sum to total project P95 (because the slowest step varies trial to trial). The total project P95 shown in Order Brief is computed independently and may be smaller than the sum of step P95s.
• Threshold bumps are applied uniformly to optimistic/most-likely/pessimistic.
